## 오늘 푼 문제

### [BOJ 14425 - 문자열 집합](https://www.acmicpc.net/problem/14425)
- 유형: Set (존재 여부)
- 핵심: 문자열 포함 여부 → HashSet.contains()
- 실수: 없음
- 배운 점: 존재 여부 판단은 고민 없이 Set 사용

### [BOJ 7785 - 회사에 있는 사람](https://www.acmicpc.net/problem/7785)
- 유형: Set (상태 관리) + 정렬
- 핵심
  - enter → add
  - leave → remove
  - 최종 상태만 유지 후 정렬
- 실수: 없음
- 배운 점
  - 정렬은 마지막에 한 번 수행
  - 자료구조 선택은 “연산 시점 vs 결과 시점” 기준으로 판단

---

## 🧠 오늘 배운 패턴
- Set
  - contains → 존재 여부
  - add/remove → 상태 관리

👉 선택 기준
- 존재 여부 → Set
- 상태 변화 → Set
- 정렬 필요 → 마지막에 처리

---

## ⚠️ 오늘 인사이트

- StringTokenizer → 빠른 입력 처리에 유리
- HashSet vs TreeSet
  - HashSet: 빠른 연산
  - TreeSet: 정렬 유지

👉 상황에 따라 선택
