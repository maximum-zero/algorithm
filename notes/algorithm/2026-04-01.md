## 오늘 배운 패턴

### 1. 슬라이딩 윈도우 (고정 길이)

핵심
- 길이가 고정된 구간 → 슬라이딩 윈도우
- 이전 값 제거(-), 새 값 추가(+)

패턴
- sum -= arr[left]
- sum += arr[right]

👉 특징
- 투포인터와 달리 "길이 고정"
- 매번 계산 ❌ → 상태 유지 ⭕

---

### 2. 구간합 (Prefix Sum)

핵심
- 반복되는 구간 조회 → 미리 계산
- O(N) 전처리 + O(1) 조회

공식
- prefix[i] = 0 ~ i-1까지 합
- 구간 [l, r] = prefix[r+1] - prefix[l]

---

### 3. 2차원 구간합 (문자 카운팅)

핵심
- prefix[c][i] = 0 ~ i-1까지 문자 c 개수
- 알파벳별 상태를 따로 관리

👉 이유
- 쿼리마다 for문 → 시간초과
- 상태를 미리 저장

---

## 실수

1. count 초기값 1로 시작
   → 첫 값이 최대가 아닐 경우 문제 발생

2. 슬라이딩 윈도우에서 조건 분기 꼬임
   → max 갱신 / count 갱신 분리 필요

3. 단순 반복으로 해결 시도 (16139)
   → 시간복잡도 고려 부족

---

## 핵심

1. 슬라이딩 윈도우
   → "고정 구간 + 상태 유지"

2. 구간합
   → "미리 계산해서 빠르게 조회"

3. 선택 기준
   → 고정 길이 → 슬라이딩 윈도우  
   → 반복 쿼리 → 구간합
