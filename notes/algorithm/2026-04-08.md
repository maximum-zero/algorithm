## 오늘 푼 문제

### [BOJ 10816 - 숫자 카드 2](https://www.acmicpc.net/problem/10816)
- 유형: HashMap (빈도수 카운팅)
- 핵심: 숫자 → 등장 횟수 저장 후 O(1) 조회
- 대안: 정렬 + 이분탐색 (lower/upper bound)
- 실수: 없음
- 배운 점: 카운트 문제는 Map이 기본이지만, 이분탐색으로도 해결 가능

### [BOJ 1920 - 수 찾기](https://www.acmicpc.net/problem/1920)
- 유형: 이분탐색
- 핵심: 정렬 후 target 존재 여부를 O(log N)으로 탐색
- 실수: 이분탐색 + 선형탐색 혼합 (비효율)
- 원인: mid 갱신 없이 left++로 순회
- 교정: while(left <= right) 구조로 절반씩 제거하도록 수정

---

## 🧠 오늘 배운 패턴
- HashMap: 빈도수 / 빠른 조회
- 이분탐색: 정렬 기반 탐색 / 존재 여부 / 범위 탐색

👉 자료구조 선택 기준
- 빠른 조회 필요 → Map
- 정렬 기반 탐색 → 이분탐색

## ⚠️ 오늘 실수

- 이분탐색 구조를 정확히 이해하지 않고 부분적으로 구현
