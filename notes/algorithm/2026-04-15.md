## 오늘 푼 문제

### [BOJ 2470 - 두 용액](https://www.acmicpc.net/problem/2470)
- 유형: 투포인터
- 핵심
  - 양 끝에서 시작 (left, right)
  - 합이 0에 가까운 값 찾기
  - sum > 0 → right--
  - sum < 0 → left++
  - 최솟값 갱신 → Math.abs()
- 실수
  - 비교 조건 반대로 작성
  - 초기 (left, right) 조합 제외
- 배운 점
  - 투포인터는 “평가 → 이동” 순서

### [PRG 43238 - 입국심사](https://school.programmers.co.kr/learn/courses/30/lessons/43238)
- 유형: 이분탐색 (Parametric Search)
- 핵심
  - 시간(mid)을 기준으로 처리 인원 계산
  - total >= n → 가능
  - 가능하면 더 줄이기 (최소값 탐색)
- 실수
  - right 범위 잘못 설정
  - isCheck 로직 이해 부족
- 배운 점
  - 이분탐색은 “정답을 찾는 게 아니라 검증하는 것”
  - mid는 정답이 아니라 테스트 값
  - 단조성(시간 ↑ → 처리량 ↑)이 핵심
