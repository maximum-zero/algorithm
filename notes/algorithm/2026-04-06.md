## 오늘 배운 패턴

### 1. 자료구조 → 배열로 변환하는 사고

핵심
- Queue / Deque 문제를 그대로 풀지 않는다
- 배열로 변환해서 “연속 구간 문제”로 바꾼다

패턴
- queue1 + queue2 → 하나의 배열
- 이후 투포인터 / 슬라이딩 윈도우 적용

👉 특징
- 자료구조 시뮬레이션보다 훨씬 단순하고 안정적

---

### 2. 배열 복사 / 확장 (System.arraycopy)

핵심
- 배열을 이어붙이거나 확장할 때 사용

문법
- System.arraycopy(원본, 시작, 대상, 시작, 길이)

패턴
```java
class Example {
   public static int[] solution(String[] gems) {
      System.arraycopy(queue1, 0, arr, 0, n);
      System.arraycopy(queue2, 0, arr, n, n);       
   }
}
```

👉 특징
- 반복문보다 빠르고 실수 줄임
- “배열 합치기” 문제에서 자주 사용

---

### 3. Map.size()로 “종류 개수” 판단

핵심
- 현재 구간에 포함된 “종류 수”를 빠르게 확인

패턴
- map.size() == 전체 종류 수

👉 특징
- 매번 전체 검사 → 시간초과
- 상태 유지 → O(N) 해결

---

### 4. map.equals vs size 비교

구분
- map.equals → key + value 완전히 동일해야 함
- size 비교 → “종류가 모두 포함됐는지”만 확인

👉 정리
- 종류 포함 여부 → size
- 완전 동일 비교 → equals

---

## 실수

1. int 사용 → overflow 발생
   - 합 관련은 long type 고려
2. 길이 계산 실패
   - right - left + 1

---

## 핵심

1. 자료구조 문제는 배열로 변환 가능성 먼저 보기
2. 상태 관리는 Map으로 유지
3. 조건 만족 시 반드시 “줄이기”
