## 오늘 배운 패턴

### 1. 슬라이딩 윈도우 (가변 길이)

핵심
- 길이가 고정되지 않음
- 조건을 만족하는 동안 확장 및 조건이 깨지면 축소

패턴
- 조건 만족 → right++
- 조건 깨짐 → left--
- 상태 유지

👉 특징
- "조건 유지"가 핵심
- 윈도우 크기가 계속 변함 

---

### 2. 문자열 슬라이딩 윈도우 (diff)

핵심
- 전체 비교 X
- 상태 변화 추적 O

개념
- diff: 서로 다른 문자 개수

패턴
- 조건 깨짐 → diff++
- 조건 만족 → diff--

공식
- 매번 비교 O(N) X → O(1)로 최적화
- "값 비교"가 아닌 "상태 변화 추적"

---

### 3. 슬라이딩 윈도우 + 덱 (최솟값)

핵심
- 최솟값이 될 수 없는 값은 제거

패턴
1. 뒤에서 큰 값을 제거
2. 현재 값 추가 (뒤에 추가)
3. 범위 벗어난 값 제거 (앞에 제거)
4. 덱의 앞은 항상 최솟값을 유지

👉 특징
- 덱은 항상 오름차순 유지
- 덱의 앞은 항상 최솟값을 유지

---

## 실수

1. 슬라이딩 윈도우 개념 혼동
   → 고정 길이 == 슬라이딩 윈도우로 접근하려고 함

2. diff 개념 이해 부족
   → 값 비교로 생각하여 기존의 kindCount와 동일한 형태로 풀이

3. 최솟값 문제 접근 실수
   → 매번 전체 탐색 (stream 의 min 메서드 사용) → 시간 부족

---

## 핵심

1. 슬라이딩 윈도우는 하나가 아님
   → 고정 / 가변 / 문자열 / 덱

2. 상태 유지의 본질
   → 전체를 다시 계산하지 않음

3. diff
   → "같냐 / 다르냐"만 추적
