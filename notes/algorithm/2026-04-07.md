## 오늘 푼 문제

### [BOJ 20920 - 영단어 암기는 괴로워](https://www.acmicpc.net/problem/20920)
- 유형: HashMap + 정렬
- 핵심: 빈도수 → 정렬 기준 설계
- 실수: 문자열 사전순 비교를 charAt() 처리 (오답 가능)
- 교정: compareTo() 문자열 비교로 변경

### [BOJ 1620 - 나는야 포켓몬 마스터 이다솜](https://www.acmicpc.net/problem/1620)
- 유형: 배열 + HashMap
- 핵심: O(1) 조회 구조 설계
- 실수: 숫자로 조회 → Map 이용하여 처리하려고 함
- 교정: 숫자는 배열 index 사용으로 변경

### [BOJ 1764 - 듣보잡](https://www.acmicpc.net/problem/1764)
- 유형: Set (교집합)
- 핵심: contains() 기반 교집합 처리
- 실수: 기준이 되는 듣도 못한 사람을 TreeSet 사용하여 TreeSet.contains() 사용 (비효율)
- 교정: HashSet (조회) + TreeSet (정렬)

---

## 🧠 오늘 배운 패턴
- Map: 값 ↔ 값 매핑 / 빈도수
- Set: 존재 여부 / 교집합
- 배열: index 기반 빠른 접근 (O(1))

👉 자료구조 선택 기준
- contains 많다 → Set
- get 많다 → Map
- index 접근 → 배열
