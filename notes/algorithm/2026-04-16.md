## 오늘 푼 문제

### [PRG 43236 - 징검다리](https://school.programmers.co.kr/learn/courses/30/lessons/43236)
- 유형: 이분탐색 (Parametric Search)
- 핵심 
  - “돌 사이 최소 거리”를 기준으로 가능 여부 판단 
  - 거리 < mid → 돌 제거 
  - 제거 개수 > n → 불가능 
  - 가능하면 더 큰 거리 탐색
- 실수 
  - mid의 의미를 이해 못해서 구현이 막힘
- 배운 점
  - 이분탐색은 “배치”가 아니라 “조건 만족 여부 판단” 
  - 정답을 직접 구하는 게 아니라 “불가능한 영역 제거”

### [PRG 135807 - 숫자 카드 나누기](https://school.programmers.co.kr/learn/courses/30/lessons/135807)
- 유형: 수학 (GCD - 유클리드 호제법)
- 핵심 
  - 배열 전체를 나누는 값 = GCD 
  - GCD가 상대 배열을 나누면 탈락 
  - 두 배열 각각 검사 후 최대값 선택
- 실수 
  - 약수를 전부 구하려고 접근함 (비효율)
- 배운 점 
  - “전체를 나눈다” → GCD 하나로 정리 가능 
  - GCD가 안 되면 나머지 약수도 전부 불가능

---

## 🧠 오늘 배운 패턴
- Parametric Search 
  - “조건을 만족하는 최소/최대 값 찾기” 
  - 가능 / 불가능 기준으로 탐색
- GCD (유클리드 호제법)
  - 공통으로 나누는 값 → GCD 하나로 압축 
  - 반복적으로 나머지 연산으로 계산

---

## ⚠️ 오늘 인사이트
- 이분탐색은 “정답을 맞추는 문제”가 아니라 “정답이 아닌 구간을 제거하는 문제”
- 문제를 보면
  - “값을 구하라” → 구현
  - “조건을 만족하는 최대/최소” → 이분탐색 의심
- 수학 문제에서 “전체를 만족” → GCD / LCM 먼저 의심
