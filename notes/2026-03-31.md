## 오늘 배운 패턴

### 1. 슬라이딩 윈도우
- 고정 길이 k 구간 유지
- remove → add 방식으로 상태 갱신

핵심 흐름
- remove (왼쪽 제거)
- add (오른쪽 추가)
- 상태 유지

👉 투포인터와 차이
- 투포인터: 범위를 늘리고 줄임
- 슬라이딩 윈도우: 범위 고정 + 상태 유지

---

### 2. 상태 유지 (핵심)
- 매번 전체 검사 ❌
- 상태를 변수로 유지 ⭕

패턴
- 조건 만족 → +1
- 조건 깨짐 → -1

예시
- 초밥 → kindCount (서로 다른 종류 개수)
- DNA → needCount (조건 만족 개수)

---

### 3. 조건 → 상태 변환 (중요)
- 조건을 매번 검사하지 않고 "카운트"로 변환

핵심
- if 만족 → count++
- if 깨짐 → count--

👉 성능 개선 핵심 포인트

---

### 4. 원형 배열 처리
- (index % N) 사용

핵심 공식
- add 위치: (i + k - 1) % N

---

### 5. 링 버퍼 (최적화)
- 전체 배열 대신 k 크기만 유지
- 배열을 큐처럼 사용

핵심
- window[idx] = 값
- idx = (idx + 1) % k

👉 메모리: O(N) → O(k)

---

## 실수

1. boolean 배열 사용
   → 중복 개수 관리 불가 → count 배열 필요

2. 인덱스 실수
   → i + k - 1 공식 놓침
   → remove / add 타이밍 헷갈림

3. 문제 해석 실수
   → 설명 vs 입력 조건 혼동
   → 불필요한 예외 처리

4. 상태 유지 개념 부족
   → 매번 검사하려고 접근

5. 링 버퍼 구현 실수
   → 원형 처리 누락
   → i 조건 헷갈림 (i >= k, i >= k-1)

---

## 핵심 깨달음

1. 슬라이딩 윈도우 본질
   → "구간 계산"이 아니라 "상태 유지"

2. 사고 방식 변화
   → 값 중심 ❌
   → 상태 중심 ⭕

3. 공통 패턴
   → add → 상태 변화
   → remove → 상태 복구
