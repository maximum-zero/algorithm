## 오늘 푼 문제

### [프로그래머스 - 조건에 맞는 사용자 정보 조회하기](https://school.programmers.co.kr/learn/courses/30/lessons/164670)
- 유형: JOIN + GROUP BY + HAVING + 문자열 처리
- 핵심
  - 사용자별 게시글 수 집계
  - HAVING으로 조건 필터
  - 전화번호 포맷 (SUBSTRING)
- 실수
  - 전화번호 컬럼 누락
  - GROUP BY 의미 없이 사용
- 배운 점
  - GROUP BY는 “집계 목적”일 때만 사용

[풀이]
```sql
SELECT
    UGU.USER_ID,
    UGU.NICKNAME,
    CONCAT(UGU.CITY, ' ', UGU.STREET_ADDRESS1, ' ', UGU.STREET_ADDRESS2) AS 전체주소,
    CONCAT(SUBSTRING(UGU.TLNO, 1, 3), '-', SUBSTRING(UGU.TLNO, 4, 4), '-', SUBSTRING(UGU.TLNO, 8, 4)) AS 전화번호
FROM USED_GOODS_BOARD UGB
    INNER JOIN USED_GOODS_USER UGU ON UGB.WRITER_ID = UGU.USER_ID
GROUP BY UGU.USER_ID
HAVING COUNT(*) >= 3
ORDER BY UGU.USER_ID DESC;
```

### [프로그래머스 - 조회수가 가장 많은 중고거래 게시판의 첨부파일 조회하기](https://school.programmers.co.kr/learn/courses/30/lessons/164671)
- 유형: JOIN + 서브쿼리 + 문자열 처리
- 핵심
  - MAX(VIEWS)로 조회수 최대 게시글 찾기
  - 해당 게시글들의 파일 조회
  - CONCAT으로 파일 경로 생성
- 실수
  - LIMIT로 1개만 선택하려고 함
  - 조건을 잘못 해석
- 배운 점
  - 최댓값 문제는 “여러 개 가능성” 고려해야 함

[풀이]
```sql
SELECT
    CONCAT('/home/grep/src/', B.BOARD_ID, '/', F.FILE_ID, F.FILE_NAME, F.FILE_EXT) AS FILE_PATH
FROM USED_GOODS_BOARD B
    INNER JOIN USED_GOODS_FILE F ON B.BOARD_ID = F.BOARD_ID
WHERE VIEWS = (
    SELECT
          MAX(VIEWS)
    FROM USED_GOODS_BOARD
)
ORDER BY F.FILE_ID DESC;
```
