## 오늘 푼 문제

### [프로그래머스 - 있었는데요 없었습니다](https://school.programmers.co.kr/learn/courses/30/lessons/59043)
- 유형: JOIN + 조건 비교
- 핵심
  - 입양일(OUTS)보다 보호 시작일(INS)이 늦은 데이터 찾기
    - AINS.DATETIME > AOUTS.DATETIME
    - DATEDIFF(AINS.DATETIME, AOUTS.DATETIME) > 0
- 실수: 없음

[풀이]
```sql
SELECT
  AINS.ANIMAL_ID,
  AINS.NAME
FROM ANIMAL_INS AINS
    INNER JOIN ANIMAL_OUTS AOUTS ON AINS.ANIMAL_ID = AOUTS.ANIMAL_ID
WHERE AINS.DATETIME > AOUTS.DATETIME
ORDER BY AINS.DATETIME;
```

### [프로그래머스 - 보호소에서 중성화한 동물](https://school.programmers.co.kr/learn/courses/30/lessons/59045)
- 유형: JOIN + 조건 + LIKE
- 핵심
  - 보호 시작: Intact 상태
  - 입양 후: Spayed 또는 Neutered
  - 상태 변화 조건 필터링
- 실수: 없음
- 배운 점
  - 상태 변화 문제 → “이전 상태 vs 이후 상태 비교”

[풀이]
```sql
SELECT
  AINS.ANIMAL_ID,
  AINS.ANIMAL_TYPE,
  AINS.NAME
FROM ANIMAL_INS AINS
    INNER JOIN ANIMAL_OUTS AOUTS ON AINS.ANIMAL_ID = AOUTS.ANIMAL_ID
WHERE AINS.SEX_UPON_INTAKE LIKE 'Intact%'
    AND (AOUTS.SEX_UPON_OUTCOME LIKE 'Spayed%' OR AOUTS.SEX_UPON_OUTCOME LIKE 'Neutered%')
ORDER BY AINS.ANIMAL_ID;
```
