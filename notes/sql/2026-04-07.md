## 오늘 푼 문제

### 가격이 제일 비싼 식품의 정보 출력하기 (프로그래머스)
- 유형: 정렬 (ORDER BY + LIMIT)
- 핵심: 최댓값 → 정렬 후 상위 1개 선택
- 대안: MAX() + Sub Query 가능

[풀이]
```sql
SELECT
    PRODUCT_ID,
    PRODUCT_NAME,
    PRODUCT_CD,
    CATEGORY,
    PRICE
FROM FOOD_PRODUCT
ORDER BY PRICE DESC
LIMIT 1;
```

### 조건에 맞는 사용자와 총 거래금액 조회하기 (프로그래머스)
- 유형: JOIN + GROUP BY + HAVING
- 핵심
  - WHERE: 집계 전 필터링 (STATUS = 'DONE')
  - GROUP BY: 사용자별 집계
  - HAVING: 집계 결과 필터링 (700_000 이상)
- 주의: GROUP BY는 SELECT 컬럼 기준으로 맞추는 것이 안전

[풀이]
```sql
SELECT
    GU.USER_ID,
    GU.NICKNAME,
    SUM(GB.PRICE) AS TOTAL_SALES
FROM USED_GOODS_BOARD GB
     INNER JOIN USED_GOODS_USER GU ON GB.WRITER_ID = GU.USER_ID
WHERE GB.STATUS = 'DONE'
GROUP BY GU.USER_ID
HAVING TOTAL_SALES >= 700000
ORDER BY TOTAL_SALES ASC;
```
