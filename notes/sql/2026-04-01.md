## 문제 1
### 조건에 맞는 사용자와 총 거래금액 조회하기 (프로그래머스)

[유형]
- JOIN + GROUP BY + HAVING

[풀이]
```sql
SELECT
    u.USER_ID,
    u.NICKNAME,
    SUM(b.PRICE) AS TOTAL_SALES
FROM USED_GOODS_BOARD b
INNER JOIN USED_GOODS_USER u
ON b.WRITER_ID = u.USER_ID
WHERE b.STATUS = 'DONE'
GROUP BY u.USER_ID, u.NICKNAME
HAVING SUM(b.PRICE) >= 700000
ORDER BY TOTAL_SALES;
```

[실수]
- GROUP BY에 불필요한 컬럼 포함 (STATUS 등)
- HAVING에 일반 조건을 넣으려 한 실수
- GROUP BY 기준을 명확히 잡지 못함

[핵심]
- WHERE: 집계 전 필터 
- HAVING: 집계 후 필터 
- GROUP BY는 “집계 기준”이다


## 문제2
### 주문량이 많은 아이스크림들 조회하기 (프로그래머스)

[유형]
- JOIN + GROUP BY + SUM + ORDER BY

[풀이]
```sql
SELECT
    fh.FLAVOR
FROM FIRST_HALF fh
LEFT JOIN JULY j
ON fh.FLAVOR = j.FLAVOR
GROUP BY fh.FLAVOR
ORDER BY SUM(fh.TOTAL_ORDER) + COALESCE(SUM(j.TOTAL_ORDER), 0) DESC
LIMIT 3;
```

[실수]
- SHIPMENT_ID로 JOIN 시도 (잘못된 기준)
- GROUP BY를 두 컬럼으로 잡으려 한 실수 
- SUM 없이 단순 덧셈으로 처리하려 함 
- INNER JOIN 사용 시 누락 케이스 발생

[핵심]
- JOIN 기준은 “문제에서 묶고 싶은 기준” 
- 집계는 항상 SUM으로 처리 
- LEFT JOIN + COALESCE로 누락 데이터 대응
